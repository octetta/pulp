the point

        
A sound tool small enough to crash, restart, script, abuse, and understand.

        
This is not a product pitch. It is a tour through a little machine built around terse text, persistent state, isolated execution, and immediate sound.

      


      

        
the stack

        
SKODE talks to Skred. Skred drives Pulp. Pulp fills the speakers.

        

          
SKODEASCII shorthand for changing sound without ceremony.

          
SkredThe shell/runtime you can run native or in a browser.

          
PulpThe C audio engine: voices, buffers, tables, parameters.

          
SpeakersThe point where architecture becomes air pressure.

why ports?

        
I do not want the audio thread to become a bureaucracy.

        
So safety lives at the process boundary. Let the sharp little engine stay sharp, then contain it with pipes, ports, browser sandboxes, and restartable runtimes.

Stateful execution, stateless-looking text.

k-synth

        
Array code is a shortcut from math to timbre.

        
Instead of hand-writing loops and tables, generate useful sound material from compact array expressions, then feed those tables into the same runtime.
